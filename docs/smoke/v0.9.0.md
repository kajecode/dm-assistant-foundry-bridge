# Smoke Test — bridge v0.9.0

**Release:** [v0.9.0](https://github.com/kajecode/dm-assistant-foundry-bridge/releases/tag/v0.9.0) · **Date:** 2026-05-16

Completes dm-assistant#503 — the bridge now translates the
`stats.spellcasting` block (dm-a v0.31.0 / contract 0.5.1) onto the
dnd5e Actor: spellcasting ability + NPC Spellcaster Level. Spell save
DC / attack / slots then compute correctly on the sheet.

**`min-api-contract-version` stays 0.5.0** — degrades cleanly against
older dm-a (every actor = non-caster). Full spellcasting needs
dm-assistant ≥ v0.31.0.

Sections:

- A — Install + version chip
- B — Spellcaster imports with ability + caster level
- C — Non-caster unaffected
- D — Graceful degrade against dm-a < v0.31.0
- E — Regression: other stat fields unaffected

## Prerequisites

- dm-assistant **≥ v0.31.0** on the Pi (contract 0.5.1) for §B/§C/§E.
- A spellcaster NPC whose `_stats.yaml` has `spellcasting:
  {ability,level}` populated — re-extract per the dm-a v0.31.0 smoke
  (`tools/extract_stats.py --campaign <id> --slug <s> --force`).
- A plain non-caster NPC.
- bridge v0.9.0 in Foundry v13 + dnd5e v5.x.

## B. Spellcaster → ability + caster level on the sheet

1. **Actors** tab → import the spellcaster NPC.

### B — Expected

- On the actor sheet's spellcasting config: the **spellcasting
  ability** matches the Stats Block prose (e.g. Intelligence) and the
  **NPC Spellcaster Level** matches (e.g. 9).
- Spell save DC + spell attack bonus are now non-default (they derive
  from the ability mod + proficiency).
- F12:

  ```js
  const a = game.actors.getName("<spellcaster>");
  console.log(a.system.attributes.spellcasting, a.system.details.spellLevel);
  // e.g. "int" 9
  ```

## Known limits (not bugs)

- **Which spells** a caster knows/prepares is the separate
  spell-list round-trip (named-spell `Item`s already come via the
  actions sidecar / #32 compendium resolution) — this release only
  sets the *ability + level* so the DC/attack/slot math is right.
- **dnd5e v5 paths.** Maps to `system.attributes.spellcasting` +
  `system.details.spellLevel`. If a future dnd5e major relocates
  these, the translator needs a follow-up (call it out if the sheet
  shows blank despite F12 showing the values).
- **Half / incoherent input collapses to non-caster.** An ability
  with level 0 (or vice-versa), or an unknown ability, is treated as
  non-caster rather than emitting a broken half state.

## If something fails

- Capture: dm-a version + contract, the spellcaster's `_stats.yaml`
  `spellcasting:` block, the actor `system.attributes.spellcasting` +
  `system.details.spellLevel` (F12 `toObject()`), the Stats Block
  prose.
- File against `kajecode/dm-assistant-foundry-bridge` with the v0.9.0
  tag.
